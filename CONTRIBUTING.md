# Contributing

## Before opening a change

- Keep the repository focused on small standalone shell utilities.
- Do not add coursework instructions, autograder files, starter code metadata, or grading artifacts.
- Do not add secrets, tokens, private identifiers, or personal data.

## Change guidelines

- Prefer portable Bash where practical.
- Keep command-line behavior explicit.
- Update `README.md` and `docs/usage.md` when script usage changes.
- Add examples only when they are verified against actual script behavior.

## Validation

If you run checks locally, record the exact commands and outcomes in the pull request description. Do not claim tests passed unless they were actually executed.

## Ownership and publication

If any script originated in a group or course context, confirm publication permission and attribution before contributing related material.
